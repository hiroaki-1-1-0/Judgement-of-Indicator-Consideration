# 大喜利相対評価時の指標考慮を判定するWebアプリ

大喜利回答を相対評価した際の文章において、評価指標が考慮されているか否かを人手評価するためのフォームを、Google Apps Script製のWebアプリとして公開し、回答内容をGoogleスプレッドシートに記録するためのレポジトリです。

完成版は[こちら](https://script.google.com/macros/s/AKfycbyLARm_Q-AICqAyHQhgett1mY_f0aipok9X-73OwvNkAmWC-fF-Yn8tIljUqCmPYzxGeg/exec)になります。

アンケートを目的としたフォームを自作したい場合に、こちらのレポジトリをご活用いただければ幸いです。

## ファイル構成

- `Code.gs`：サーバー側処理。Googleスプレッドシートへの保存、保存先シートの初期化を行います。
- `Index.html`：フォーム画面です。左側に四つ組データ、右側に11指標の選択欄を表示します。
- `Data.html`：CSVから生成した33件の評価対象データと指標名です。`Index.html` から読み込まれます。

## 表示されるデータ

左側：

1. 大喜利のお題
2. 大喜利の回答1
3. 大喜利の回答2
4. 相対評価時の文章

右側：

1. 語彙的新規性の低さ
2. 不一致の解消度合い
3. 比喩の使用度
4. 連想距離の遠さ
5. ボケの面白さ
6. 曖昧さの活用度
7. 無害な違反の度合い
8. 視点の転換度合い
9. 特殊文字の少なさ
10. お題と回答の文字数比率の低さ
11. 文字数の少なさ

各指標では、次のいずれかを選択します。

- 考慮している
- 考慮していない
- 考慮しているか不明

`相対評価時の文章` は、見出し・太字・箇条書き・区切り線・引用・インラインコードなどの簡易マークダウン処理を施して表示します。

## 設定手順

### 1. 記録先Googleスプレッドシートを作成する

1. Googleドライブで新しいGoogleスプレッドシートを作成します。
2. URLを確認します。URLは次のような形式です。

```text
https://docs.google.com/spreadsheets/d/スプレッドシートID/edit#gid=0
```

3. `/d/` と `/edit` の間にある文字列がスプレッドシートIDです。これを控えます。

### 2. Apps Scriptプロジェクトを作成する

1. 作成したスプレッドシートを開きます。
2. メニューから **拡張機能 → Apps Script** を開きます。
3. 既存の `コード.gs` があれば中身を削除し、このパッケージの `Code.gs` の内容を貼り付けます。
4. `Code.gs` 冒頭の次の部分を、先ほど控えたスプレッドシートIDに置き換えます。

```javascript
const SPREADSHEET_ID = 'ここに記録先スプレッドシートIDを貼り付けてください';
```

例：

```javascript
const SPREADSHEET_ID = '1abcDEFghiJKLmnOPqrSTuvWXyz1234567890';
```

### 3. HTMLファイルを作成する

1. Apps Scriptエディタ左側の **＋** を押します。
2. **HTML** を選択します。
3. ファイル名を `Index` にします。拡張子 `.html` は自動で付きます。
4. 作成された `Index.html` の中身をすべて削除し、このパッケージの `Index.html` の内容を貼り付けます。
5. 同じ手順でHTMLファイルをもう1つ作成し、ファイル名を `Data` にします。
6. 作成された `Data.html` の中身をすべて削除し、このパッケージの `Data.html` の内容を貼り付けます。

### 4. 回答保存用シートを初期化する

1. Apps Scriptエディタ上部の関数選択で `setupSheet` を選びます。
2. **実行** を押します。
3. 初回はGoogleの認可画面が出ます。自分のアカウントで許可してください。
4. スプレッドシートに `responses` というシートが作成され、ヘッダー行が入ります。

注意：`setupSheet` は既存の `responses` シートを初期化します。運用開始後に実行すると保存済み回答が消えるため、最初の1回だけ実行してください。

### 5. ウェブアプリとしてデプロイする

1. Apps Scriptエディタ右上の **デプロイ → 新しいデプロイ** を押します。
2. 種類の選択で **ウェブアプリ** を選びます。
3. 設定を次のようにします。
   - 説明：任意。例 `仮デプロイ`
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：回答者の状況に合わせて選択
     - 外部回答者に配布する場合：**全員**
     - 組織内だけの場合：組織内ユーザー
4. **デプロイ** を押します。
5. 表示されたウェブアプリURLを回答者に共有します。

### 6. 動作確認

1. デプロイ後のウェブアプリURLを開きます。
2. 名前を入力し、判定画面に進めることを確認します。
3. 1件分について11指標すべてを選択し、「保存して次へ」を押します。
4. スプレッドシートの `responses` シートに1行追加されていることを確認します。

## 注意事項

- `setupSheet` は最初の1回だけ実行してください。
- デプロイ後にコードを修正した場合は、**デプロイ → デプロイを管理 → 編集 → 新バージョン** で再デプロイしてください。
- 回答者が戻るボタンで前の項目に戻った場合、画面上の選択状態は復元されます。ただし、すでに保存済みの行をスプレッドシート上で上書きする仕様ではありません。運用時は「保存して次へ」を1回ずつ押して進める案内にしてください。


## 今回の保存値仕様

右側の各評価項目は、スプレッドシートに以下の数値で保存されます。

- `考慮している` → `1`
- `考慮していない` → `-1`
- `考慮しているか不明` → `0`

この仕様は主に `Index.html` の `CHOICES` と `collectRatings()`、および `Code.gs` の `ALLOWED_VALUES` / `saveResponse()` で制御しています。
